**ultrathink**
変更差分に対してレビュー→検証→修正のループを実行する。

## ループフロー

以下を問題が0件になるまで繰り返す。最大5回でループを打ち切る。

### 1. レビュー実行

変更ファイルを確認する。

**各コマンドは個別に実行すること。`&&` で連結しない。**

```bash
git diff --name-only HEAD
```

```bash
git diff --name-only --cached
```

#### CodeRabbit CLI（常に実行）

```bash
cr review
```

`cr` が利用できない場合はスキップし、レポートにその旨を記載する。

#### カスタムレビュアー

`.claude/vibecorp.yml` の `review.custom_commands` を確認する。定義がある場合、各コマンドを並列で実行する。

### 2. 妥当性検証

レビュー結果を検証し、以下を判定する:

- **修正すべき指摘**: コード品質・正確性に関わる妥当な指摘
- **却下する指摘**: 好みの問題、過剰な最適化、文脈を誤解した指摘

却下する場合は理由を明記する。

**要修正0件ならループ終了。**

### 3. 修正計画

要修正リストに対して、各指摘の具体的な修正計画を策定する:

- 対象ファイルと変更箇所
- 修正内容の概要
- 副作用の有無

### 4. 修正実行

修正計画に従って修正を適用する。

### 5. 結果報告（ループ終了後）

全イテレーションの結果をまとめて報告する:

```text
## review-loop 結果

### 修正した指摘
- {ファイル}: {指摘内容} → {修正内容}

### 却下した指摘
- {指摘内容} — 理由: {却下理由}

### サマリ
- ループ回数: {n}回
- 修正: {n}件
- 却下: {n}件
- 最終レビュー: 問題0件
```

## 制約

- `--force`、`--hard`、`--no-verify` は使用しない
- 最大5回でループを打ち切る（無限ループ防止）
- 修正前に必ず関連ファイルを読み込む
