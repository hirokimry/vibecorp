# Context7 ドキュメント取得スキル

Context7 CLI (`c7`) を使い、ライブラリ・フレームワークの最新ドキュメントを取得する。
エージェントが古い知識やハルシネーションに基づいてコードを書くリスクを軽減する。

## 使用方法

```bash
/vibecorp:context7 <ライブラリ名>
/vibecorp:context7 <ライブラリ名> --tokens <トークン数>
```

- `<ライブラリ名>`: ドキュメントを取得したいライブラリ名（例: `react`, `nextjs`, `tailwindcss`）
- `--tokens <トークン数>`: 取得するドキュメントの最大トークン数（デフォルト: 5000）

## worktree モード

`--worktree <path>` が指定された場合、全操作を指定パス内で実行する。

- **Bash**: 全コマンドを `cd <path> && command` で実行する
- 未指定時は従来通り CWD で実行する（後方互換）

## ワークフロー

### 1. Context7 CLI の存在確認

```bash
command -v c7
```

`c7` が見つからない場合、以下のメッセージを出力して**スキルを終了**する:

```text
## Context7 CLI が見つかりません

Context7 CLI (`c7`) がインストールされていません。以下の手順でインストールしてください:

### インストール方法

npm install -g @upstash/vibecorp:context7

### 参考

- https://github.com/upstash/vibecorp:context7

インストール後、再度 `/vibecorp:context7 <ライブラリ名>` を実行してください。
```

### 2. ライブラリ ID の解決

```bash
c7 resolve "<ライブラリ名>"
```

出力から候補一覧を取得する。候補が複数ある場合は最も関連性の高いもの（先頭）を選択する。

候補が見つからない場合:

```text
「<ライブラリ名>」に一致するライブラリが見つかりませんでした。
ライブラリ名を確認して再度実行してください。
```

### 3. ドキュメントの取得

```bash
c7 get <ライブラリID> --tokens <トークン数>
```

- `--tokens` 未指定時はデフォルト 5000 を使用する
- コマンドがエラーを返した場合、エラー内容を表示して終了する

### 4. 結果の出力

取得したドキュメントを以下の形式で出力する:

```text
## Context7: <ライブラリ名> のドキュメント

**ソース**: Context7 CLI (`c7 get <ライブラリID>`)
**トークン数**: <実際に取得したトークン数>

---

<取得したドキュメント内容>
```

## エラーハンドリング

| エラー | 対応 |
|---|---|
| `c7` 未インストール | インストール手順を案内してスキル終了 |
| ネットワークエラー | 「ネットワーク接続を確認してください」と表示 |
| ライブラリ未発見 | 「ライブラリ名を確認してください」と表示 |
| API エラー | エラーメッセージをそのまま表示 |
