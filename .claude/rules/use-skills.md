# ⚙️ スキル使用義務

> [!IMPORTANT]
> 定義済みのスキル（`/vibecorp:ship` / `/vibecorp:ship-parallel` / `/vibecorp:review-loop` / `/vibecorp:commit` 等）がある操作は、**手動でステップを再現せず必ずスキルを使う**。
> スキルのステップを **部分的に手動実行しない**。レビューループ / マージ待ちが欠落する原因になる。
> PR 作成で完了とせず、スキルが定義する **全ステップ**（レビュー / CI 確認 / auto-merge 設定）を実行する。

本ルールはスキル（動線）/ エージェント（専門家）/ ルール（常駐規約）の役割境界を運用面で強制する。スキルが定義する手順を手動でつまみ食いすると、責務境界が崩壊する。

## 🎯 対象範囲

| 対象 | 適用 |
|------|------|
| 主 Claude（COO）の操作 | ✅ 適用する |
| Agent ツールで起動した子エージェントの操作 | ✅ 適用する |
| ヘッドレス（`claude -p`）起動の操作 | ✅ 適用する |
| 並列 ship（`/vibecorp:ship-parallel`）の各 worktree | ✅ 適用する |

## ✅ 指針（MUST）

- ✅ 定義済みスキルがある操作は **必ずスキルを呼ぶ**（手動再現しない）
- ✅ Agent ツールで子エージェントに作業を委譲する場合も、子エージェントに **スキルの使用を指示する**
- ✅ スキルが定義する **全ステップを完遂する**（PR 作成だけで終わらせない）
- ✅ レビューループ / CI 確認 / auto-merge 設定までセットで実行する

## ❌ 禁止パターン

- ❌ スキルのステップを **部分的に手動実行する**（レビューループやマージ待ちが欠落する）
- ❌ PR 作成で完了として、後続のレビュー / CI / auto-merge を省略する
- ❌ 子エージェントに「スキルを使わず手動で実装してください」と指示する
- ❌ 「同じことを手で書けるから」を理由に `git` / `gh` を直接叩いてスキルを迂回する

## 📋 主要スキル動線

| 操作 | 必ず使うスキル |
|------|-------------|
| Issue → PR | `/vibecorp:ship` |
| 並列複数 Issue | `/vibecorp:ship-parallel` |
| PR レビュー対応 | `/vibecorp:review-loop` / `/vibecorp:pr-fix-loop` |
| コミット | `/vibecorp:commit` |
| PR 作成・更新 | `/vibecorp:pr` |
| Issue 起票 | `/vibecorp:issue` |
| ブランチ作成 | `/vibecorp:branch` |
