# 🌟 MVV 適用ルール

> [!IMPORTANT]
> 全ての判断・レビュー・提案は `MVV.md` に定義された **ミッション・ビジョン・バリュー** に基づく。
> MVV に反する変更は、他のルールに適合していても **却下** する。
> エージェントによる `MVV.md` 本体の編集・改変提案は **禁止**。

本ルールは vibecorp の根本方針である MVV を全エージェントの判断軸として固定する。`vibecorp.yml` の `protected_files: MVV.md` で本体は書込み保護されており、本ルールはその意思決定面の遵守を補強する。

## 🎯 対象範囲

| 対象 | 適用 |
|------|------|
| C\*O / SM / 平社員エージェントの判断 | ✅ 適用する |
| Issue / PR レビュー | ✅ 適用する |
| スキル内のオーケストレーション判断 | ✅ 適用する |
| `/diagnose` の改善候補抽出 | ✅ 適用する |
| 仕様策定・新規機能提案 | ✅ 適用する |

## ✅ 指針（MUST）

- ✅ 判断・レビュー・提案の前に `MVV.md` を参照する
- ✅ 他のルールと衝突した場合は MVV を優先する
- ✅ 判断に迷った場合は MVV に立ち返る
- ✅ MVV を判断根拠として引用する場合は該当節を明示する

## ❌ 禁止パターン

- ❌ MVV に反する変更を「他ルールに適合している」を理由に承認する
- ❌ `MVV.md` 本体をエージェントが直接編集する
- ❌ `MVV.md` 本体の改変提案を CEO 未承認のまま起票する
- ❌ MVV を参照せずに新規機能・大規模変更を判断する

## 👤 編集権限

`MVV.md` 本体の編集はファウンダー（CEO）のみが行う。

- 編集経路: CEO の直接編集のみ
- エージェント経路: 一切認めない（読み取り専用）
- 保護機構: `vibecorp.yml` の `protected_files` + `protect-files.sh` フック
