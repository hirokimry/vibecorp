# コミュニケーション規約

CEO への対話応答・Issue 本文・PR 本文・コミットメッセージ・監査レポートは、本規約に従って書くこと。

## 前提

CEO は代表取締役社長＝経営者である（根拠: `docs/ai-organization.md` の基本思想、`.claude/rules/roles.md` の CEO 定義）。
CEO は「ソフトウェアが何をできるようになったか／壊れたか／変わるか」で判断する。エージェントの役目は、その判断材料を最短で渡すことにある。

## 対象文面

本規約は以下の文面に適用する。

- CEO への対話応答（エージェントがユーザーに直接返すテキスト）
- Issue タイトル / 本文
- PR タイトル / 本文
- コミットメッセージ
- 監査レポート（`/audit-cost`, `/audit-security` など）

## 対象外（正確性優先で良い）

以下は正確性・網羅性を優先し、本規約の「動作主語」ルールを適用しない。

- `.claude/rules/` のエージェント向けルール（本ファイル含む）
- コード内コメント
- `docs/` の技術仕様書
- プランファイル（`~/.cache/vibecorp/plans/**`）

## 1. 動作主語で語る

- **主語**は「このソフトウェア／このフック／このスキル／このエージェント」にする
- **時制**は「〜になった／〜できるようになった／〜しなくなった／〜するようになった」を使う
- コード参照が必要な場合は、動作説明の **後ろ** に `📍 根拠:` で最小限に添える

### Before / After

| 文面 | Before（コード主語・NG） | After（動作主語・OK） |
|---|---|---|
| Issue | `knowledge_buffer.sh` の `flush_buffer()` を修正 | ✅ 知見バッファが PR 作成後に自動で空になるようになった |
| PR | `protect-files.sh` の正規表現を変更 | 🔒 `settings.json` を誤って編集しようとすると自動で止まるようになった |
| 応答 | `get_config()` を置換 | ⚠️ 設定取得が未設定時に落ちていた → デフォルト値を返すようになった |

## 2. マークダウンと状態絵文字

見出し・箇条書き・テーブルで一覧性を出し、以下の絵文字で状態を示す。

| 絵文字 | 意味 |
|--------|------|
| ✅ | 完了・成功 |
| ⚠️ | 注意・警告 |
| ❌ | 失敗・却下 |
| 🚀 | リリース・デプロイ |
| 🔒 | セキュリティ |
| 🐛 | バグ |
| ✨ | 新機能 |
| 📖 | ドキュメント |
| 🔄 | リファクタ |
| 🧪 | テスト |
| 📍 | 根拠・位置参照 |
| 🤖 | エージェント |

### 30 秒ルール

CEO が 30 秒読んで「何が変わるか」を掴めたら合格。掴めなければ書き直す。

## 3. 判定フロー

文面を書く前・書いた後に以下を通す。

1. **読み手は CEO か？** → Yes なら本規約を適用（No なら対象外として正確性優先）
2. **主語が動作になっているか？** → ソフトウェアやフック等の挙動を主語にしているか
3. **30 秒で掴めるか？** → マークダウン整形と状態絵文字で変化が一目で分かるか

3 つ全てを満たしたら OK。満たさなければ書き直す。
