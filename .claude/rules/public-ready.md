# Public Ready ガードレール

## 1. セキュリティ・機密情報の混入禁止

- シークレット、トークン、APIキー、パスワードを絶対にコミットしない
- 個人情報（メールアドレス、本名、社内URL等）を含めない
- .env ファイルや credentials をコミットしない
- git history に上記が残ることも許容しない（後から消しても履歴に残る）

## 2. 特定プロダクト・リポジトリへの参照禁止

- 特定の別リポジトリやプロダクト名を直接記載しない
- 「既存プロジェクトから移植した」等の由来を示す記述をコードやコミットメッセージに残さない
- コメントや TODO に特定プロジェクト名を書かない
- 汎用的な表現（「導入先プロジェクト」「対象リポジトリ」等）を使う

## 3. ローカル環境・ユーザー設定への依存禁止

- ~/.claude/ 等のユーザーグローバル設定に依存しない
- 特定のマシンパスをハードコードしない（~/dev/..., /Users/xxx/... 等）
- 環境変数で吸収する場合は、デフォルト値を持たせるか、未設定時に明確なエラーを出す
- 動作に必要な前提条件は README に明記する

## 4. git history の品質

- コミットメッセージは公開されても恥ずかしくない内容にする
- WIP、仮置き、デバッグ用コードをコミットしない
- 意味のある単位でコミットする（巨大な一括コミットを避ける）

## 5. テスト必須

- シェルスクリプト（install.sh, hooks等）は必ず自動テストを書く
- テストは `tests/` ディレクトリに配置する
- push 前にテストが通ることを確認する
- テストなしのコードを push しない
