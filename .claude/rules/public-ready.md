# 🌐 Public Ready ガードレール

> [!IMPORTANT]
> vibecorp は Public 公開前提のリポジトリ。**全てのコミットがそのまま公開されても問題ない** 状態を維持する。
> シークレット / 個人情報 / 特定別リポ参照 / ローカル依存 を絶対にコミットしない。
> git history に残ったら手遅れ（後から消しても履歴に残る）。

本ルールは Public 公開時の事故を未然に防ぐガードレール。5 領域（セキュリティ / 別リポ参照 / ローカル依存 / git history / テスト）を常時遵守する。

## 🎯 対象範囲

全ファイル（`paths: ["**/*"]`）。コミット対象になる全てを審査対象とする。

## 🔒 1. セキュリティ・機密情報の混入禁止

| 種別 | 禁止内容 |
|------|---------|
| シークレット | API トークン / 暗号鍵 / パスワード / OAuth secret |
| 個人情報 | 個人メールアドレス / 本名 / 社内 URL |
| 認証ファイル | `.env` / `credentials.json` / `~/.aws/credentials` 等の中身 |
| git history | 上記が一度でも commit に入ったら **後から消しても履歴に残る** ため許容しない |

- ❌ シークレット / トークン / API キー / パスワードを絶対にコミットしない
- ❌ 個人情報（メールアドレス / 本名 / 社内 URL 等）を含めない
- ❌ `.env` ファイル / credentials をコミットしない

## 🚫 2. 特定プロダクト・リポジトリへの参照禁止

| 種別 | 禁止 / 推奨 |
|------|------------|
| 特定別リポ名 / プロダクト名 | ❌ コード / コミットメッセージ / コメント / TODO に書かない |
| 由来表記 | ❌ 「既存プロジェクトから移植した」等を残さない |
| 汎用表現 | ✅ 「導入先プロジェクト」「対象リポジトリ」を使う |

## 🏠 3. ローカル環境・ユーザー設定への依存禁止

- ❌ `~/.claude/` 等のユーザーグローバル設定に依存する
- ❌ 特定マシンパス（`~/dev/...` / `/Users/xxx/...` 等）をハードコードする
- ✅ 環境変数で吸収する場合は、**デフォルト値を持たせる** か、未設定時に **明確なエラーを出す**
- ✅ 動作に必要な前提条件は `README` に明記する

## 📜 4. git history の品質

| 項目 | 内容 |
|------|------|
| コミットメッセージ | 公開されても恥ずかしくない内容にする |
| 仮置きコード | WIP / 仮置き / デバッグ用コードをコミットしない |
| 粒度 | 意味のある単位でコミットする（巨大な一括コミットを避ける） |

## 🧪 5. テスト必須

- ✅ シェルスクリプト（`install.sh` / hooks 等）は必ず自動テストを書く
- ✅ テストは `tests/` ディレクトリに配置する
- ✅ push 前にテストが通ることを確認する
- ❌ テストなしのコードを push しない
