# MVP と プリセットの関係を明確に分離する

## 混同しやすいポイント

初期設計では「MVPだから機能を削る」という発想で、
minimal プリセットの一部機能（スキル群、エージェント）を MVP から除外していた。

これにより「minimal プリセットを選んだのに /review が使えない」という矛盾が生まれた。

## 正しい定義

- **MVP** = vibecorp プラグインの最小出荷単位。ユーザーに「動く」と言える状態
- **preset** = 組織規模の選択肢。MVP 完了時点で少なくとも1つのプリセットが完全に動く

つまり:
- MVP = minimal プリセットが **完全に** 動作する状態
- MVP で「minimal の一部」が動く、ではない

## 教訓

「MVPだから削る」と「プリセットとして小さい構成を用意する」は別の話。
MVPでも選んだプリセットの機能は100%動かないとユーザー体験が破綻する。
