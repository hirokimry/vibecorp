# CFO 判断ログ

## 2026-04-11: `--dangerously-skip-permissions` コンテナ化隔離のコスト評価

### 背景

オーナーが `claude --dangerously-skip-permissions` を安全に使うため、コンテナ化による隔離構想を検討。
CFO 観点でコスト・ROI を評価した。

### コスト試算（定性評価 / 実コスト未確定）

#### 増加コスト

| 項目 | 規模感 | 備考 |
|------|--------|------|
| CPU/メモリオーバーヘッド | 小（5〜15%程度） | Docker 起動・cgroup 分離コスト |
| イメージビルド時間 | 初回 2〜5 分、キャッシュ後 30 秒以下 | layer キャッシュで吸収可 |
| CI 実行時間増加 | 中（+1〜3 分/ジョブ） | Docker-in-Docker 構成の場合に顕著 |
| ストレージ（イメージ） | 月 $1〜5 程度（GHCR 無料枠内の可能性大） | ベースイメージ 1〜2 GB 程度 |

#### セルフホスト vs. マネージド比較

| 方式 | 月額コスト感 | 主なトレードオフ |
|------|------------|----------------|
| ローカル Docker（開発者端末） | ほぼ $0 | 端末スペックに依存、CI には使えない |
| GitHub Actions + docker run | $0〜10（無料枠内） | DinD 設定コスト、CI 分速消費 |
| GitHub Codespaces | $5〜50/人/月 | 手軽だが従量課金が積み上がる |
| dev container（VS Code/Codespace） | Codespaces 準拠 | 開発体験向上とセットで評価すべき |
| 専用 VM（self-hosted runner） | $10〜30/月 | 管理コスト増、ただし CI コスト固定化できる |

#### 削減されるリスクコスト

- `--dangerously-skip-permissions` の誤操作による被害（ファイル削除、シークレット漏洩等）は最悪ケースで数時間〜数日のリカバリ工数
- 開発者 1 名 × 1 日 = $500〜2,000 相当の機会コストと想定
- 月 1 回でも事故が起きれば、コンテナ化コストは 1 ヶ月以内に回収

### 判断

**条件付き承認**

- ローカル Docker またはGitHub Actions 上の docker run 構成を推奨
- Codespaces はコスト単価が高く、個人開発・OSS フェーズでは過剰
- イメージのキャッシュ戦略（layer 分割、GHCR 活用）を設計段階で決める
- 月次コストレビュー時に CI 分速消費を確認する項目として追加すること

### ROI 見立て

初期セットアップコスト（エンジニア工数 0.5〜1 日）と月次ランニングコスト（$0〜10）に対し、
事故 1 件回避で元が取れる。ROI はポジティブ。ただし Codespaces をフル活用する場合は
利用人数・頻度を見て月次で費用対効果を再評価すること。
